1. What's the difference b/w Div tag & Span tag?

Div Tag:

Div is a division tag which we used for dividing the content while we creating the larger projects/Websites. It can also be used to group elements for styling, layout or scripting purpose. It's a Block line Element.

Span Tag:

It's used to group and style elements within a document. It doesn't inherently provide any visual changes, but by using this tag, we can apply CSS & Javascript to specific portions of text. It is an Inline Element.

3. What's the difference b/w href & src attribute?
href:

href means hyper-referrence is an attribute which we used to create a links to other pages or resources. It specifies a link to another resource, which will be opened in a new page or tab based on the value's we given when clicked.

It's commonly used with Anchor tags.

src attribute:

src means Source attribute which is used to embed the content in current webpage. It specifies the source of an external resource that will be embedded directly in to the current page. It's commonly used with image tag, iframe tag, audio tag, video tag, script tag.

4. What's the difference b/w HTML & XML
HTML:
  1. HTML is Hyper text markup language.
  2. We used for structure and display the content on webpages.
  3. Carries data to be displayed to the user.
  4. less flexibility in terms of defining custom structure.

XML:
  1. XML means Extensible markup language
  2. It's used for to transport and store the data.

5. Difference b/w the push and pul commands in git?

Git Push and Git Pull are two essential commands for syncing changes between your local repository and a remote repository.

  1. Git push Command purpose is Uploads local changes to a remote repository.
    Direction: Local ---> Remote
  2. We use Git Push when you want to share your code with others.


  1. Git pull command purpose is to Downloads the changes from a remote repository and merges them into your local branch.
    Direction: Remote ---> Local
  2. Use Git Pull when you want to update your local repository with changes from others.

7. What are the stages of our project to reach to github repo?

There are 4 stages that our project to reach to github repo. They are:

  1. Working Directory: In this, we initiate the new repo in local repo and then add the files.
  2. Staging area: In this, we commit the files and create the branches or checkout the branches and check the status.
  3. Local Repo: In this, we push our project from our local repo to Github repo.
  4. Github Repo: In this, once we push the project from our local repo the files will be added to the github repo. From there, we can deploy the code or Project.
